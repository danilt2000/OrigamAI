 This text will show you how you can bind two entities using a foreign key (FK). 
 A foreign key is used to link two relational database tables. It is a field or a group of fields in one table that uniquely identifies a row of another table (primary key). 
 
 [!success]- Prerequisites for further work 
In order to create a foreign key, there are 2 basic prerequisites you need to meet: 
 
  Have a running database server  
  Have a project created in ORIGAM Architect  
 
 
 In the ORIGAM Architect, there are two ways of how to create a foreign key – using an action (wizard) and manually. 
   Naming convention 
 The convention of ORIGAM is that to the basic FK field name, we add  ref  (reference) a prefix and  Id  (identifier) as suffix. So, for example, if we have an entity named  Task  and a field named  Priority  in it in which we want to store the Id of the priority stored in the table  Priorities , we name that field  refPriorityId . 
 This naming convention helps to recognize at the first look that such a field contains a foreign key and is linked to a certain record, a primary key, of another entity. We don’t use any underscore-type hyphens, because that would complicate things later on when the field name becomes part of a larger name string, for example in the database ( FK_Task_refAssignedToId_BusinessPartner ) or in the XSLT transformations etc. 
   Foreign key with Action 
 When using Action, the wizard will create a new database field for you and then make it a foreign key. If you want to use the wizard, this field must not yet exist. 
 To do that, just right-click in the Model browser on the entity for which you want to create the field with FK and choose  Action / Create Foreign Key… :
