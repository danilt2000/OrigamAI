   Actions 
 To create workflows more quickly, you can use some predefined actions over the context store: 
  


[Image: https://community.origam.com/uploads/default/original/2X/e/e23c06671291e1f626b4d008f63feab5c3da09c1.png]

Selecting "Actions" reveals a submenu with options: "New LoadData Task," "New StoreData Task," "New Transform Task," and "Load/Transform/Save Tasks." This interface appears designed to manage data workflow tasks efficiently.


  
 Here, you can either select individual steps for working with data — load, transform, and store — or an entire sequence comprising all these model steps. This will pre-generate a three-step workflow for you:

   Fine-tuning your workflow 
   Workflow step labels 
 To make the second step, where the user selects tasks, more user-friendly, change the workflow step label. 
 Proceed as follows: 
 
 Select the step in the diagram editor of Model Browser 
 Go to the  Documentation  tab (accessible from the main Menu under  View ) 
 Under  Category , select  USER_WFSTEP_DESCRIPTION  
 In the Text field, enter for example “Select tasks” — this is what users will see in the client application 
 Click the save icon
