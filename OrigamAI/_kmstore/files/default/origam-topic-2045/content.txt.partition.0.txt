# Complex Data Rule

 An  XSLT  based rule used for enforcing rules on data (calculated fields). 
 Input context: Full document 
 It is added as a normal data structure rule with the difference that the  TargetField  attribute is left empty.

 Then you add a new Complex Data Rule and in the Settings panel define the destination data structure where the transformed data will be merged.

 If you want to also apply a condition rule in the data structure rule pointing to a complex data rule, the attribute  IsPathRelative  must be set to  False  and the  XPath  expression contain the full path, e.g.  /ROOT/element/@attribute .
