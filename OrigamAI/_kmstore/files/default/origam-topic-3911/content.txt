 This article will show you how to create new database entities and fields in the ORIGAM Architect and then how to project your work into the application’s database. 
 An  Entity  is the most basic part of the  Data Model . A database entity maps to a database and represents one database table. Every  Field  represents one column in the table. 
 Both Entities and Fields, as well as other objects in the platform, get by default a unique identifier (GUID) -  a long unique hex string named  Id :

Description: The image depicts a user interface for a database entity editor labeled "Priority." In the visible section labeled "Info," key fields are presented, including "Id" which contains the value "5ff8e826-c960-414c-9bf7-743a4acdb1bd." Other fields include "ItemType" designated as "DataEntity," "PackageName" identified as "myapplication," "RelativeFilePath" specifying "myapplication\DataEntity\myapplication\Priority.origam," and "SchemaItemType" as "Origam.Schema.EntityModel.TableMappingItem." The purpose of this interface is to manage and display detailed information about a specific database entity within a given application.


  
 By this Id the records are referenced for example using Foreign Keys.

  Have a running database server  
  Have a project created in ORIGAM Architect

  Name  
  Caption  
  DataType  
  DataLength  (where needed, for example, by String type)

   Deployment to database 
 Any new database entities and fields you create need to be reflected in your database. In ORIGAM Architect, this is a two-step process. 
 First, run the Deployment Script Generator in Menu / Tools:

Description: The image depicts the user interface of ORIGAM Architect, specifically showcasing the menu options available under the "Tools" menu in the context of a model application. The currently highlighted option is "Deployment Script Generator," with additional options visible including "Restart Server(s)," "SQL Console," "Show Web Application," "Generate GUID" (with the shortcut Ctrl+Shift+G), and "Dump Window XML." The interface features typical menu items for managing application deployment and server operations, indicating a focus on application configuration and management tasks. The application version shown is 1.0.0.
