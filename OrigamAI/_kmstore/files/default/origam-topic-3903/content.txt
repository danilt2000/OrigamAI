# How to create a new Project

URL: https://community.origam.com/t/how-to-create-a-new-project/3903

@vikter (2025-01-13 09:02:39Z):
   
 This tutorial will guide you step by step through the process of setting up a new project in ORIGAM Architect. 
 
 [!success]- Prerequisites for further work 
To proceed, you need: 
 
  A running database server  
 
 
   Run the wizard 
 You can find the option to create a new project – that is, a new application or system – in the main menu under File.

The apparent purpose of this interface is to provide users with various functionalities related to project management and connection settings within the ORIGAM Architect environment.


  
 When you launch Architect for the first time, the wizard for creating a new project will start automatically. 
   Name your project 
 Enter a name for your project. Be sure to choose the name carefully, as changing it later will be very challenging. 
  


[Image: https://community.origam.com/uploads/default/original/2X/1/1ea78ccf9e98c44e258d1abd98882210f0f4edd0.png]
Description: The interface is a dialog window titled "New Project." It focuses on entering a project name, indicated by the label "Project Name." It will be used to set the package, database and folders names. Example: MyNewProject." At the bottom right, there are two buttons labeled "Next" and "Cancel," suggesting a sequential workflow for project creation. The purpose of the dialog is to set up a new project by collecting the initial name input.


  
   Connect the database 
 The next step is to configure the database connection. In this example, we will use an MS SQL server running locally in a Docker container. 
  


[Image: https://community.origam.com/uploads/default/original/2X/e/e0a90c44b0331fe8548a98198e903804e83040c2.png]
Description: The image depicts a user interface for creating a new project involving database configuration, labeled "New Project" at the top. The section titled "Database" contains form fields for the following: a dropdown for selecting the "Database" type, pre-filled with "Microsoft Sql Server," a text input for "Database Server" set to "localhost," and a "Port" field set to "1433." Additional fields include "User Name," with "sa" filled in, and a "Password" field represented with asterisks. Below the fields, there are instructional texts advising to enter the database user's name and password, and that the user must have create database privileges. There are buttons labeled "Next" and "Cancel" at the bottom right.


  
 You need to enter the username and password that you created when  setting up your database server . If these details are incorrect, you will not be able to complete the creation of the new project. 
 
 
Want to use PostgreSQL? 
 In that case, select PostgreSQL as the Database and then later in the already created project, edit the configuration in OrigamSettings.config - Data Connection, both  DataConnectionString  and  DataDataService , plus  DeployPlatforms . In addition to that you need to set the DatabaseType property in the Docker .env file to “postgresql“.

 In this step, you will create an account for the first user in your future application.

   Save your model 
 At this stage, Architect will create a  Git repository  for your project and configure your username and email for it. 
 If you do not wish to create a repository, simply uncheck this option. 
  


[Image: https://community.origam.com/uploads/default/original/2X/b/b8ccd692334577beb6811f1d201bc605e0567058.png]
Description: The image displays a user interface titled "New Project" under the section labeled "Model Location." It includes a checkbox labeled "Create GIT repository." A note states, "A new folder with your project's name will be created here. Origam will store the model in it." At the bottom, there are two buttons labeled "Next" and "Cancel," indicating the next steps in project creation.


  
 You will also need to choose the location for your project model. 
 You can leave the project folder as is or select your own. Since we will build on this setup in future tutorials, it is recommended to keep the default settings. 
   Complete the process 
 Now, just run the final step, and your project will be created. If your database connection is correctly configured, the process should finish successfully.

 If any issues arise, use the arrow above to return to the appropriate step, adjust the settings, and run the final process again.
